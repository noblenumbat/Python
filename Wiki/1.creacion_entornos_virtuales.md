# USO DE ENTORNO VIRTUAL PARA NO CONTAMINAR EL PYTHON DEL SISTEMA

1. 
```
python3 -m venv mi_entorno
source mi_entorno/bin/activate
```

> [!Note]
> En mi caso Mint no tenia el paquete python3.12-venv 
>
> Intente la instalación con:  
```apt install python3.12.3-venv``` 
>
> pero surgió un error ... error code (1)

**COMO LO SOLUCIONE**

1. Reparando la base de datos de dpkg 

```
sudo dpkg --configure -a
sudo apt install -f
```

2. Sincronizando y actualizando los repositorios:

```
sudo apt update && sudo apt upgrade
```

3. Instalando la versión de manera automática para que coincidida con la versión del sistema:

```
sudo apt install python3-venv
```

**COMO SE QUE MI ENTORNO ESTA ACTIVO**

Porque aparece algo como esto: ```entorno_librerias_python_joncas)``` que precede el promt en la terminal.

**SIGNIFICADO** 
+ Que cualquier libreria que yo instale com pip install se guardara solo dentro de esa carpeta y no afectará al Python principal en mi sistema.

+ En cuanto a rutas el comando python ahora apunta al ejecutable que esta dentro de la carpeta de entorno, ignorando versiones globales del sistema.

**EN QUE CASOS Y COMO SE USA**

Se usa siempre que se este trabajando en un proyecto especifico.
+ **Para instalar librerias:** si necesito algo como pandas o request, lo instalo mientras el entorno está activo: ```pip install pandas```
+ **Para ejecutar código:** corro archivos .py sabiendo que tienen acceso a esas librerías instaladas.
+ **Para salir:** para cuando termine de programar por hoy, ejecutar ```deactivate```para salir del entorno.

**SE ACTIVA CADA VEZ QUE SE VAYA A USAR**

```source nombre_entorno/bin/activate```

> [!Note]
>
> Es recomendable crear uno por cada proyecto diferente.
